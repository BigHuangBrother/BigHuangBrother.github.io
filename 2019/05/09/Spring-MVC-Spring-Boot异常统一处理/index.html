Spring MVC/Spring Boot异常统一处理

##背景
前几天在公司某个系统中服务请求失败，奇怪的是Console确没有打印异常信息。作者一直以为是logback配置文件有问题，可能异常信息级别不够或者输出位置不对。
看了很久发现logback配置文件是没问题的。后来联想到可能对异常做了统一处理，终于发现有个类继承了WebMvcConfigurerAdapter类重写了configureHandlerExceptionResolvers方法
对异常做了统一处理。

可见代码中只对一种异常做了处理，其他异常都忽略掉了，所以才导致其他异常没有打印。

##总结
其实应该早就想到有一个统一异常处理类。借此在这复习一下，做下总结.

Spring 统一异常处理有 3 种方式，分别为：

  1. 使用 @ ExceptionHandler 注解
  2. 实现 HandlerExceptionResolver 接口
  3. 使用 @controlleradvice 注解
    ###使用 @ ExceptionHandler 注解 可以看到，这种方式最大的缺陷就是不能全局控制异常。每个类都要写一遍。

###使用 @ControllerAdvice+ @ ExceptionHandler 注解
上文说到 @ ExceptionHandler 需要进行异常处理的方法必须与出错的方法在同一个Controller里面。那么当代码加入了 @ControllerAdvice，则不需要必须在同一个 controller 中了。这也是 Spring 3.2 带来的新特性。从名字上可以看出大体意思是控制器增强。 也就是说，@controlleradvice + @ ExceptionHandler 也可以实现全局的异常捕捉。
